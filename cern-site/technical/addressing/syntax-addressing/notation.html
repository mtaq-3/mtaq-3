Notation

This section covers the notational conventions used in WWW specifications, specifically an augmented Backus–Naur Form (BNF). It is divided into three parts: standard RFC 822 conventions, specific changes for WWW, and general rules used across the project.

Notational Conventions (BNF)

  1. Rule Naming
    • Rule names are written plainly, such as RuleName.
    • Literal text is enclosed in quotation marks.
  2. Alternatives
    • Elements separated by | represent alternatives, e.g. foo | bar.
    • The vertical bar is used instead of / because slashes appear frequently in directory paths.
  3. Repetition
    • * indicates repetition.
    • <l>*<m>element means at least l and at most m occurrences.
    • 1*element requires at least one occurrence.
  4. Specific Repetition
    • Exactly n occurrences are written as <n>(element).
    • Example: 2DIGIT is a two‑digit number.
  5. Lists
    • The # construct indicates a comma‑separated list.
    • 1#element requires at least one non‑null element.
  6. Comments
    • A semicolon ; begins a comment that continues to the end of the line.

Changes and Additions for WWW

  • Non‑terminals are written in PascalCase, e.g. NonTerminalName.
  • A rule is written as NonTerminal ::= RuleDefinition.
  • C‑style escape sequences are used for layout characters:
    • \n (newline)
    • \r (carriage return)
    • \t (tab)
    • \b (backspace)
    • \f (form feed)
  • Angle brackets are avoided to prevent conflict with SGML/HTML tags.

General Rules

  • HTTP data is written in line format, meaning \r\n line breaks are significant.
  • Strings are sequences of ASCII characters surrounded by double quotes.
  • Double quotes inside strings must be escaped as \".
